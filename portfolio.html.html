Continuous Trackers / Models

Politics

The i UK Poll Tracker, (Project, Code). Scraping: Python (Selenium). Model: R (brms package) [July 2021] [The i]

A Poll tracker for The i that uses a Bayesian Dirichlet model to estimate the latent voter intention, with confidence intervals, for all major parties up until the latest poll. The model also includes dynamically weighted house effects and time variables. It is currently the only poll tracker from a UK-based paper that attempts to model voter intent rather than using a rolling average.

2020 US Election Model, (Project, Code). Back-end: R, STAN. Front-end: d3.js, JavaScript, html, CSS. [November 2020] [Other]

This is a state-space model to predict the result of the 2020 election, designed as a STAN implementation of Drew Linzer's 2008 paper on the subject, with correlated priors. The model correctly predicted one less state than The Economist's + 538. I wrote a full breakdown of every model's performance here.

Covid-19

The i Covid-19 Tracker. (Project, Code). Python, R [June 2021] [The i]

I built a Python wrapper for the Covid-19 dashboard API v2 (documentation only existed for v1). I used this with data from Our World in Data to supply live interactive Datawrapper charts that are used across The i (350,000 views in the last month). The data also supplies my Covid-19 tracker visuals, which are built in R and generated daily. R was chosen instead of d3js as there is currently no way of integrating d3 into The i's CMS.

Covid-19 Test tracker (deprecated). (Code). Python (Selenium, Bs4, Pandas), RegEx [May 2020] [Other]

A tracker for Covid-19 tests implemented at the start of the pandemic. It used a mix of BeautifulSoup, Selenium and RegEx to track test numbers from 33 different countries. The project was discontinued when I started my Masters.

Interactive projects

Politics

Trump's Tweets, (Project, Code) d3js, JavaScript, html, CSS [January 2021] [Other]

A graphic display of all of Trump's tweets since his 2016 candidacy announcement to his Twitter suspension, accounting for favourites, retweets, and whether the tweet was later deleted or flagged. The visualisation features a tooltip that dynamically wrangles the base data using RegEx so that it can be rendered in the style of a typical tweet.

Economics

What Happened to the US Treasuries in March? (Project, Code) d3js, jQuery CSS ,html [November 2020] [Other]

A scrolling narrative to explain in an elegant manner what happened to the US treasury market during the Covid-19 crisis. The scrollytelling framework which underpins this story does not rely on 3rd party libraries but was designed by hand for this project. It was built using a mixture of CSS, jQuery and d3.

Other Projects

A Study of Twitter (Project, Code) Fasttext (for NLP), Gephi (Network analysis), Front end: CSS, HTML, jQuery, Plotly [April 2020] [Other]

I designed this project as way of charting ‘British political twitter’ while also investigating which voices have the most influence on social media. I used network analysis through Gephi, along with the Twitter API, to uncover the particular communities within political Twitter. I also used Facebook’s Machine Learning library, Fasttext, to analyse the sentiment of each tweet, which I used to chart the divisiveness of particular topics.

Chexit.co.uk (Project, Code) PHP + MySQL, Ajax, JavaScript, CSS + HTML [September 2019] [Other]

During the Brexit saga, I designed this database of every MP’s vote on over 50 different Brexit divisions. It was intended as a resource through which anyone could find an easily digestible summary of an MP’s Brexit position.

Journalism

Politics

New Constituencies Map (Project) R (sf) [June 2021] [The i]

For The i's politics team I modelled the political composition of the new proposed constituencies. I did this by regressing the ward-level results from the 2021 local elections to the new constituencies through geospatial analysis and the simple features package for R. I picked up on new potential gains for Greens, something which was later corroborated by Ben Walker in the New Statesman.

Facebook Ads (Project, Code) Python, R [June 2021] [The i]

As part of The i's coverage of the Batley and Spen by-election, I used my access to the Facebook Ad Library to evaluate how many adverts the Conservative and Labour parties were placing in Batley and Spen ahead of the by-election. I then compared this to the level of advertising in the two previous by-elections.

Economics

Greenbelt Graph (Project, Code) R [July 2021] [Masters]

I visualised the regional inequality of the housing crisis by comparing which local authorities were and weren't building enough houses. I did this by contrasting the numbers of new dwellings to the computed government projections for each council based on this methodology.

OpenTable Analysis (Project, Code) Python (Selenium) [October 2020] [Masters]

I used Selenium to scrape data from the OpenTable website which gave me a real-time perfectly accurate picture of the number of reservations made through Opentable at every restaurant in London. I used this data to analyse how different areas of London had recovered during the break between the 1st and 2nd lockdowns.

Covid-19

Vaccines Rates in the UK(Project, Code) R (modelr for imputing linear models) [July 2021] [The i]

I examined the reopening on July 19th by analysing vaccine rates in different ways. I compared vaccine rates by deprivation level and region in a beeswarm chart and I compared the vaccine rates of upper-tier authorities to the UK national average in a faceted line graph. I also used rolling linear projections as a way of examining how the rate of the vaccine rollout has slowed in June.

Covid-19 Hospital Cases (Project, Code) R [June 2021] [The i]

I analysed how new Delta variant cases were impacting hospitalisations. I compared the rise in cases to a comparable stretch during the first wave to demonstrate how, even though hospitalisations were rising, they were doing so at a much slower rate than they did during the first wave.
